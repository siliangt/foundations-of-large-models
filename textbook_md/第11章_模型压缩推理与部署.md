# 第11章 模型压缩、推理与部署

历经了预训练与微调，我们的大模型终于做好了迎接用户的准备。然而，将一个几百 GB 大小的大模型部署到线上环境中为千万用户提供毫秒级响应，是一项极具挑战的系统工程。大模型的自回归生成特性（每一个 Token 必须等上一个 Token 生成完才能计算）导致推理过程常常面临严重的“显存容量墙”和“内存带宽墙”。

本章将聚焦于让大模型变得“更小、更快”的核心技术。我们将探讨模型压缩的艺术（量化、剪枝与蒸馏），深度解析大模型推理特有的 KV Cache 机制及其革命性优化，并了解现代推理框架与端侧部署的未来。

## 11.1 模型量化：低比特存储的魔法

在深度学习中，模型参数默认以 32 位浮点数（FP32）或 16 位浮点数（FP16/BF16）格式存储。**量化（Quantization）**旨在将高精度的浮点数，映射到低精度的整数（如 INT8、INT4 甚至更低），从而在极少牺牲甚至不牺牲准确率的前提下，大幅削减显存占用并加速推理。

### 11.1.1 PTQ（训练后量化）与 QAT（量化感知训练）
- **训练后量化（Post-Training Quantization, PTQ）**：在模型微调结束后直接应用量化算法。针对大模型的经典算法有 GPTQ 和 AWQ。大模型中存在一些极端的激活值（Outliers），AWQ 通过保留 1% 包含最重要激活值的权重不量化（或对其进行尺度缩放），能极其完美地在 INT4 精度下维持原有性能。
- **量化感知训练（Quantization-Aware Training, QAT）**：在训练或微调的过程中模拟量化误差，让模型去适应这种低精度环境。上一章提到的 QLoRA 就可以视为一种高级的 QAT 变体。

通过量化，一个 7B 模型可以从 14GB 压缩到不到 5GB，直接塞进消费级显卡甚至手机中。

## 11.2 剪枝与知识蒸馏简介

除了量化，压缩大模型体积还有两大经典策略，尽管它们在超大模型上的应用难度远高于量化。

### 11.2.1 剪枝（Pruning）
剪枝的思想是直接“剪去”模型中不重要、或者接近 0 的连接权重甚至整个神经元结构。分为非结构化剪枝（零散剪掉神经元，很难获得硬件加速）和结构化剪枝（直接剪掉某个注意力头或某一整层网络）。大模型由于极度依赖其庞大的参数量，激进的剪枝极易导致性能断崖式下跌。

### 11.2.2 知识蒸馏（Knowledge Distillation）
这是一种“师徒传授”模式。我们用一个庞大的、极度聪明的“教师模型（Teacher）”（如 GPT-4）去指导一个小型的“学生模型（Student）”（如 LLaMA-7B）训练。学生不仅要学习真实的标签，还要学习教师模型输出的软标签（概率分布）。在开源社区，大量小模型通过模仿 GPT-4 产生的指令问答数据（如 Alpaca 数据集）来进行微调，这本质上就是一种黑盒知识蒸馏。

## 11.3 核心推理加速：KV Cache 机制解析

要理解大模型的推理加速，必须理解自回归生成的最大痛点，以及解决这个痛点的神兵利器——**KV Cache**。

### 11.3.1 什么是 KV Cache？
当模型生成一段长文本时，如果每次生成下一个词，都把之前生成的所有词重新放进 Transformer 里算一遍 $Q, K, V$，就会产生极其庞大的重复计算量。
因为每一个历史 Token 的 $K$ 和 $V$ 向量，在未来的生成中是绝对固定且不再改变的。因此，**KV Cache** 的思想是：把历史 Token 计算好的 Key 和 Value 向量全部缓存（Cache）在显存里。每次只需计算最新那一个词的 Query，去和缓存里的 Key 计算注意力得分即可。
这把时间复杂度从 $O(N^2)$ 骤降到了 $O(N)$，是大模型推理的绝对基石。

### 11.3.2 PagedAttention 与显存碎片革命
尽管 KV Cache 极大加速了计算，但它是一个贪婪的“显存吞噬者”。尤其在长文本并发请求（Batching）时，KV Cache 的显存占用随生成的长度动态增长。
传统的分配方式会预先为每个请求分配最大长度的连续显存，导致高达 60%~80% 的显存碎片浪费。
加州大学伯克利分校的研究团队从操作系统底层的虚拟内存分页中获得灵感，提出了 **PagedAttention** 技术。它将 KV Cache 切分为固定大小的“块（Block）”，允许非连续地存储在显存中。这彻底消灭了显存碎片，将大模型的并发吞吐量提升了数十倍。

## 11.4 推理框架与服务化部署（vLLM、TGI）

伴随着 PagedAttention 的诞生，一系列极度高效的大模型推理框架在工业界迅速普及：

- **vLLM**：PagedAttention 的发源地，当前开源界最火爆的高吞吐量推理引擎。它极大优化了批处理（Continuous Batching），使得单卡能够同时服务数百个请求。
- **TGI（Text Generation Inference）**：由 HuggingFace 官方推出的推理框架，集成了大量尖端技术（如 Tensor Parallelism、FlashAttention），被大量应用于生产环境。
- **TensorRT-LLM**：NVIDIA 官方推出的大模型推理库，针对自家 GPU 底层指令集进行了极限的深度优化。

## 11.5 边缘侧部署与端侧大模型展望

随着量化技术的成熟和手机、PC 等硬件算力的不断提升，大模型的战场不再仅仅局限于云端机房。
**端侧部署（Edge AI / On-Device AI）** 正在成为新的风口（如运行在笔记本上的 LLaMA.cpp 框架，以及内嵌在智能手机中的百亿级大模型）。
端侧大模型带来的优势是显而易见的：**极致的隐私保护、无需网络的极低延迟、以及无需昂贵云端算力的免费使用**。未来，端云协同的大模型架构将重塑整个人机交互的范式。
